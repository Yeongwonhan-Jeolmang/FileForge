# Contributing to FileForge

Thank you for your interest in contributing to FileForge. We welcome bug reports, feature requests, documentation improvements, and code contributions.

## How to contribute

1. Fork the repository.
2. Create a branch for your change: `git checkout -b feature/short-description`.
3. Make your changes and test locally.
4. Commit with a clear message.
5. Open a pull request against `main`.

## Issue reporting

- Use the bug report template for defects.
- Use the feature request template for enhancements.
- Include clear reproduction steps, expected behavior, and environment details.

## Code style

- Keep code readable and consistent with the existing style.
- Use descriptive names for variables, functions, and classes.
- Add comments for non-obvious logic.

## Pull request process

- Keep each pull request focused on a single topic.
- Add a summary of the change and any testing performed.
- Ensure the project still builds after your changes.
- Link related issues when applicable.

## Testing

- Run `python -m py_compile` on modified Python files.
- If you add new functionality, update the documentation as needed.
